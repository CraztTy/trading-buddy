# 舆情 / 叙事平行轨（`track:narrative`）

> 与 **[TREND_ITERATION_BACKLOG.md](TREND_ITERATION_BACKLOG.md)** 中 **「趋势轨」** 分离：**不纳入趋势 v0 验收**；单独开 issue 时使用标签 **`track:narrative`**（可再加 **`later`** 表示本季度不做）。

---

## 1. 范围（写什么算本轨）

| 类型 | 示例 | 与趋势轨关系 |
|------|------|----------------|
| **新闻 / 公告** | 抓取或接入摘要、情感打分、事件标签 | 可与日 K 对齐做研究，**验收口径单开** |
| **社媒 / 讨论度** | 热度、情绪指数（须注意合规与 ToS） | 同上 |
| **外部事件日历** | 宏观数据发布、财报密集期 | 可作为回测分层变量，**非价量 MVP 必需** |
| **预测市场等** | Polymarket 类盘口（若产品需要） | **只读展示或研究**起步；不涉及本仓库交易执行 |

---

## 2. 合规与产品边界（写进 PR / Issue 描述）

- **不做**「无人工审核的自动荐股」或保证收益类表述；叙事信号默认 **研究用途**。  
- **数据来源**：写明提供商、延迟、覆盖范围、是否可商用；保留抓取/拉取日志或哈希便于审计。  
- **与 A 股行情混用**：若做对齐，须文档化 **时区、交易日 vs 自然日、对齐键（code / 主题 ID）**。

---

## 3. 建议分期（可改成你们自己的里程碑）

| 阶段 | 目标 | 产出物（示例） |
|------|------|----------------|
| **N0** | 定边界与标签 | 本文件 + Issue 模板一句「`track:narrative`」 |
| **N1** | 单一数据源只读入库或只读 API | 一张表或一个 **`GET /api/...`** 占位 + 单测契约 |
| **N2** | 与行情弱关联 | 按 `trade_date` 或事件时间窗 join 文档 + 脚本（不必进主看板） |
| **N3** | 产品化 | 看板 Tab、权限、缓存与速率限制 |

当前仓库 **无** 叙事专用 API/表；从 **N0/N1** 起步即可。
